Create a translation management system for my psilyou platform. This is a centralized admin tool that manages translations across multiple Repl projects.

### Database (Supabase - already exists)

You'll work with two tables:

**1. translations table:**
- key (VARCHAR 255) - translation key like "dashboard.welcome_title"
- locale (VARCHAR 10) - language code: da, en, sv, de, no
- text (TEXT) - the translated content (can be word, sentence, or paragraphs)
- namespace (VARCHAR 50) - identifies which Repl/project: "my-psilyou-dashboard", "main-site", etc.
- reviewed (BOOLEAN) - false for AI drafts, true for human-reviewed
- updated_at (TIMESTAMP)
- UNIQUE constraint on (key, locale, namespace)

**2. languages table:**
- locale (VARCHAR 10) PRIMARY KEY - da, en, sv, de, no
- name (VARCHAR 50) - "Danish", "English"
- native_name (VARCHAR 50) - "Dansk", "English"
- status (VARCHAR 20) - 'draft', 'live', 'archived'
- completion_percent (INT) - auto-calculated based on reviewed translations
- display_order (INT) - sort order in language switchers
- updated_at (TIMESTAMP)

### Core Features

**1. Translation Management UI**
- View all translations filtered by: namespace, locale, reviewed status
- Edit translation text inline
- Mark translations as reviewed
- Search by key or text content
- Show completion % per language per namespace

**2. Language Management**
- Add new language (creates draft status)
- Trigger AI translation for all keys in a namespace
- View completion progress
- Promote language from draft → live when ready
- Archive unused languages

**3. AI Translation (OpenAI Integration)**
- Bulk translate: take all English (en) strings and create drafts in new language
- Single translation: translate one key on-demand
- Context-aware: pass surrounding keys for better context
- Mark all AI translations as reviewed=false

**4. API Endpoints (for other Repls to consume)**
- GET /api/languages - returns only status='live' languages
- GET /api/translations/:locale/:namespace - returns all translations for i18next consumption
- POST /api/translations/missing - log missing translation keys from client apps

**5. Analytics Dashboard**
- Show translation coverage per namespace
- List untranslated keys
- Show recently updated translations
- Track which namespaces are most complete

### Technical Stack
- Frontend: React + TypeScript with shadcn/ui components
- Backend: Express.js
- Database: Supabase (use service role key to bypass RLS)
- AI: OpenAI API for translations
- Use the same modern stack as my other psilyou Repls

### Key Requirements
- Fast API responses (cache translations, serve via CDN-friendly headers)
- Secure admin panel (basic auth or simple password protection is fine for now)
- Clean UI following psilyou brand (orange primary, blue outline secondary buttons)
- Namespace isolation - each Repl project has its own translation set
- Auto-calculate completion_percent when translations change

### Initial Data
Start with these languages in draft mode:
- da (Dansk/Danish) - primary target language
- en (English) - source language, already complete
- sv (Svenska/Swedish)
- de (Deutsch/German)

### Workflow Example
1. Admin adds Swedish language → status='draft'
2. Click "AI Translate from English" → creates all sv translations with reviewed=false
3. Admin reviews/edits Swedish translations, marks as reviewed=true
4. When 95%+ reviewed → admin promotes sv to status='live'
5. Swedish now appears in language switchers across all Repls

Build a clean, functional admin interface that makes managing translations across 5+ languages and multiple projects easy.
